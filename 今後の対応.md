## 本番稼働までに必要な対応

### 1. Supabase テーブル作成

Supabase Dashboard の SQL Editor で以下を実行:

```sql
-- supabase/migrations/002_create_shipping_label_jobs.sql の内容
CREATE TABLE IF NOT EXISTS shipping_label_jobs (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  order_id TEXT NOT NULL,
  shipping_method TEXT NOT NULL,
  status TEXT NOT NULL DEFAULT 'pending',
  result JSONB,
  error TEXT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  completed_at TIMESTAMPTZ
);

CREATE INDEX IF NOT EXISTS idx_shipping_label_jobs_status ON shipping_label_jobs(status);
CREATE INDEX IF NOT EXISTS idx_shipping_label_jobs_order_id ON shipping_label_jobs(order_id);
```

### 2. Browserless Cloud アカウント設定

- [browserless.io](https://www.browserless.io/) で無料アカウント作成
- ダッシュボードから WebSocket URL + トークンを取得
- URL 形式: `wss://production-sfo.browserless.io/chromium/playwright?token=YOUR_TOKEN`

### 3. ワーカー用環境変数の準備

以下の環境変数を `.env` ファイルまたはホスティング先のシークレットに設定:

```bash
# Browserless Cloud
BROWSERLESS_WS_ENDPOINT=wss://production-sfo.browserless.io/chromium/playwright?token=YOUR_TOKEN
PLAYWRIGHT_HEADLESS=true
PLAYWRIGHT_IGNORE_HTTPS_ERRORS=true

# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key

# Google Sheets
GOOGLE_SERVICE_ACCOUNT_BASE64=your-base64-key
GOOGLE_SHEETS_SPREADSHEET_ID=your-spreadsheet-id

# 配送サービス認証
CLICKPOST_EMAIL=your-email
CLICKPOST_PASSWORD=your-password
YAMATO_MEMBER_ID=your-member-id
YAMATO_PASSWORD=your-password

# Slack 通知（任意）
SLACK_WEBHOOK_URL=https://hooks.slack.com/services/...
```

※ Vercel の環境変数や GitHub Secrets と同じ値を使用

### 4. ワーカー起動

**Docker の場合:**
```bash
docker-compose -f docker-compose.label-worker.yml up -d
```

**Docker なしの場合:**
```bash
# .env.local に環境変数を設定して
npm run worker:labels
```

### 5. Vercel 環境変数の確認

Vercel 側にも以下が設定されていること:
- `NEXT_PUBLIC_SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`

（ジョブ投入 API とステータス確認 API が Supabase に接続するため）
