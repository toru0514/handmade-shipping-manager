# UC-003: 発送前注文一覧を表示する

## 基本情報

| 項目 | 内容 |
|-----|------|
| ユースケースID | UC-003 |
| ユースケース名 | 発送前注文一覧を表示する |
| アクター | 作家 |
| 優先度 | 高 |
| ステータス | 未実装 |

## 概要

発送待ちの注文を一覧画面で表示し、作家が発送作業を効率的に行えるようにする。

## 事前条件

- 作家がシステムにログインしていること
- スプレッドシートに注文データが存在すること

## 事後条件

- 発送前ステータスの注文一覧が画面に表示されている

## 基本フロー

1. 作家はダッシュボード画面にアクセスする
2. システムはスプレッドシートから「発送前」ステータスの注文を取得する
3. システムは注文を購入日時の古い順に並べ替える
4. システムは以下の情報を一覧表示する：
   - 注文番号
   - プラットフォーム名（アイコン表示）
   - 購入者名
   - 購入品名
   - 購入日時
   - 経過日数
   - 伝票発行ボタン（クリックポスト / 宅急便コンパクト）
5. 作家は一覧から発送したい注文を確認する

## 代替フロー

### 2a. 発送前の注文が存在しない場合
1. システムは「発送待ちの注文はありません」とメッセージを表示する

### 3a. フィルタリングを行う場合
1. 作家はプラットフォームでフィルタリングできる
2. 作家は購入日時の範囲でフィルタリングできる

## ビジネスルール

- BR-001: 注文は購入日時の古い順（FIFO）で表示する
- BR-002: 3日以上経過した注文は警告表示する
- BR-003: 発送済みの注文は一覧に表示しない

## 非機能要件

- 一覧の初期表示は2秒以内
- 100件までの注文を1ページで表示可能
